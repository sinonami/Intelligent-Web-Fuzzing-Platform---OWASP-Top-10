Getting started

Quick start

1
Paste a target URL Go to the home page and enter the full URL of the web application you want to test (e.g. https://testphp.vulnweb.com).
2
Click Run scan The scanner starts immediately. A progress bar shows each phase — crawling, fuzzing, injections, auth checks, and AI analysis.
3
Review the dashboard Results are mapped to OWASP Top 10. Click Details on any finding to see the payload, AI confidence score, and full verdict.
4
Export as PDF Click Generate PDF to download a full report with all findings, payloads, and AI verdicts — ready to share.
Coverage

What gets tested

  • SQL Injection (A03) — error-based, blind time-delay, UNION-based, boolean-based across all forms and URL parameters
  • XSS (A03) — reflected payloads tested via forms and URL params, verified by AI model
  • SSRF (A10) — internal URL injection via redirect, url, path, dest, and callback parameters
  • Path Traversal (A01) — ../ sequences and encoded variants on file/path parameters
  • Broken Access Control (A01) — exposed admin paths, sensitive file enumeration, 403/401 detection
  • Security Misconfiguration (A05) — missing headers, wildcard CORS, open config endpoints
  • Outdated Components (A06) — server version banners, X-Powered-By disclosure
  • Auth Issues (A07) — insecure cookie flags, missing rate-limiting signals
  • Logging Failures (A09) — absent CSP, HSTS, X-Frame-Options headers
Machine learning

AI classification

Every finding is passed through trained AI models before being reported. This reduces false positives and adds a confidence score to each result.

sqli_hybrid_model.h5

TensorFlow/Keras hybrid model trained on SQL injection payloads. Detects error-based and blind injection patterns.

xss_hybrid_model.h5

TensorFlow/Keras model for XSS payload classification. Verifies reflection and DOM injection signatures.

sensitive_mlp_model.pkl

MLP classifier for auth and configuration anomalies — cookie issues, missing controls, design flaws.

anomaly_config_detector.pkl

Isolation Forest model for traffic and response anomaly detection during the crawl phase.

Reports

PDF reports

After any scan completes, click Generate PDF on the results dashboard. The report includes:

  • Target URL and scan timestamp
  • Every finding with type, severity, and OWASP category
  • The exact payload that triggered each finding
  • Full AI verdict and confidence score

Reports are generated server-side and downloaded as a .pdf file named after the target domain.

Account

Scan history

All scans run while logged in are saved automatically. Visit History to see past scans with their date, target URL, and issue count.

You must be registered and signed in for scans to be saved. Anonymous scans are not stored.
