OWASP Top 10 — 2021

The ten most critical web application security risks, and how Intelligent Fuzzer tests for each one.

A01:2021

Broken Access Control

Restrictions on authenticated users are not properly enforced. Attackers can access unauthorized functionality or data.

✓ Tested — admin path enumeration, sensitive file exposure
A02:2021

Cryptographic Failures

Weak or missing cryptography exposes sensitive data. Includes cleartext transmission, weak algorithms, exposed keys.

~ Partial — private key exposure, MD5 pattern detection
A03:2021

Injection

SQL, XSS, and command injection via hostile data sent to interpreters. One of the most common and dangerous risks.

✓ Tested — SQLi (error + blind), XSS reflection, AI-classified
A04:2021

Insecure Design

Missing or ineffective security controls at the design level. Includes debug information leakage and error handling failures.

✓ Tested — stack trace and debug error disclosure detection
A05:2021

Security Misconfiguration

Misconfigured permissions, unnecessary features enabled, default accounts, verbose error messages.

✓ Tested — CORS, exposed configs, missing headers, open endpoints
A06:2021

Vulnerable & Outdated Components

Using components with known vulnerabilities. Version disclosure enables targeted attacks.

✓ Tested — Server banner, X-Powered-By version fingerprinting
A07:2021

Identification & Auth Failures

Broken authentication allows attackers to assume user identities. Includes weak sessions and missing MFA.

✓ Tested — cookie flags, rate-limit indicators, session analysis
A08:2021

Software & Data Integrity Failures

Code and infrastructure that does not protect against integrity violations, including insecure deserialization.

~ Partial — cookie attribute validation
A09:2021

Security Logging & Monitoring Failures

Insufficient logging prevents detection of breaches. Missing security headers block CSP and HSTS enforcement.

✓ Tested — CSP, HSTS, X-Frame-Options header audit
A10:2021

Server-Side Request Forgery (SSRF)

The server fetches a remote resource based on a user-supplied URL without validation.

✓ Tested — internal URL injection via redirect/url/path parameters
